# КРИТИЧЕСКОЕ ОБНОВЛЕНИЕ БЕЗОПАСНОСТИ

## CVE-2025-55182 (React2Shell) - КРИТИЧЕСКАЯ УЯЗВИМОСТЬ

### Статус: ⚠️ ВАШЕ ПРИЛОЖЕНИЕ БЫЛО УЯЗВИМО

**Дата обнаружения:** 7 декабря 2025  
**Уровень критичности:** КРИТИЧЕСКИЙ  
**CVE:** CVE-2025-55182

### Что произошло?

Обнаружена критическая уязвимость в React Server Components, которая позволяет злоумышленникам выполнять произвольный код на сервере через Server Actions.

### Ваше приложение было уязвимо потому что:

1. ✅ **Использовалась уязвимая версия React 19.1.0**
2. ✅ **Активно используются Server Actions** (`"use server"` в 20+ файлах)
3. ✅ **Next.js 15.4.4** использует React Server Components по умолчанию

### Что было исправлено:

1. ✅ **Обновлен React до версии 19.1.2** (исправляет уязвимость)
2. ✅ **Обновлен react-dom до версии 19.1.2**
3. ✅ **Добавлена защита Server Actions** (`src/shared/security/server-action-protection.ts`)
4. ✅ **Добавлена валидация входных данных** для предотвращения инъекций команд

### Что нужно сделать СРОЧНО:

#### 1. Обновить зависимости
```bash
npm install react@^19.1.2 react-dom@^19.1.2
npm install  # для обновления lock файла
```

#### 2. Пересобрать и переразвернуть приложение
```bash
npm run build
# Затем развернуть в production
```

#### 3. Проверить логи на признаки компрометации

Проверьте логи на наличие:
- Команд `wget`, `curl`, `bash`, `powershell`
- Попыток выполнения команд через `exec`, `spawn`
- Запросов к подозрительным IP-адресам
- Попыток чтения системных файлов (`/etc/passwd`, `/etc/shadow`)

#### 4. Проверить файлы конфигурации

Проверьте, не были ли изменены:
- `.env` файлы
- Конфигурационные файлы AWS/облачных сервисов
- Файлы с секретами и токенами

#### 5. Сменить все секреты и токены

Если обнаружены признаки компрометации:
- Смените все API ключи
- Смените токены аутентификации
- Проверьте AWS credentials
- Проверьте доступы к базам данных

### Признаки компрометации:

Согласно отчетам, злоумышленники:
1. Выполняют команды PowerShell для проверки уязвимости
2. Загружают скрипты с внешних серверов
3. Устанавливают бэкдоры (Cobalt Strike)
4. Проводят разведку (команды `whoami`, `id`)
5. Пытаются украсть файлы конфигурации AWS

### Дополнительные меры защиты:

1. **Rate Limiting** - добавлен в middleware
2. **Валидация входных данных** - все Server Actions теперь проверяют входные данные
3. **Мониторинг** - логирование всех подозрительных запросов
4. **Изоляция контейнера** - улучшена безопасность Docker контейнера
